## 一、产品定义

### 1.1 核心概念

**把「灵感加工系统」产品化——让用户把人生碎片（技能/爱好/习惯/能力/知识点）当拼图入库，点击「融合拼图」，AI自动完成 8 刃切割 + 灵感碰撞，输出可执行的行动方案。**

本质：将你过去 2 个月迭代出来的 linggan v5.2 认知锻造流程，封装成一个普通人能用的产品。

### 1.2 核心类比

| 现有工具 | 缺失的能力 |
|---------|-----------|
| Notion/Obsidian | 只存不连，被动等待用户自己发现联系 |
|  flomo/卡片笔记 | 碎片化输入，但没有「主动融合」的引擎 |
| ChatGPT | 对话式，没有持久的个人拼图库 |
| 第二大脑系统 | 复杂，门槛高，普通人用不起来 |

**拼图融合引擎的差异化：有记忆 + 有主动融合 + 有可执行输出 + 有成长闭环。**

### 1.3 目标用户（3类，按优先级调整）

> **v1.1调整**：第一阶段聚焦P1副业探索者，P0/P2降为Phase 2+

| 优先级 | 用户画像 | 痛点 | 我们的价值 |
|--------|---------|------|-----------|
| **P0（原P1）** | **副业探索者（想做副业但不知道从哪入手）** | 技能零散，不知道怎么组合成产品 | 拼图融合直接给出「你能做什么产品」 |
| P1（原P0） | 知识工作者（25-40岁，有笔记习惯但感觉碎片无用） | 存了很多，用不起来 | AI主动帮你「拼」 |
| P2（原P2） | 内容创作者（需要持续产出选题） | 灵感枯竭，选题重复 | 拼图碰撞自动产出新角度 |

**第一阶段聚焦P0（副业探索者）理由**：
- 痛点最明确：「想变现但不知道做什么」
- 付费意愿最强：看到可执行的副业方案愿意付费
- P1知识工作者需要"先教育市场"（二阶需求问题）
- P2内容创作者已有成熟工具链，难切入

### 1.4 核心价值主张（一句话）

> **「把你散落的能力碎片，拼成能变现的完整作品。」**

**Landing Page标题（v2.2）**：
> 你有一堆技能，但不知道怎么变现？
> AI帮你把散落的能力碎片，拼成你的组合技

### 1.5 与竞品的核心差异

- Obsidian：你要自己建链接 → 我们：AI主动帮你找链接
- Notion AI：基于当前页面向前问 → 我们：跨所有碎片全局融合
- ChatGPT：每次对话重新开始 → 我们：持久记忆你的拼图库
- **ChatGPT：无融合历史 → 我们：可视化你的融合成长轨迹**

### 1.6 与ChatGPT的差异化展示（v1.1新增）

**Landing Page新增板块：「ChatGPT vs 拼图融合引擎」**

| 功能 | ChatGPT | 拼图融合引擎 |
|------|---------|-------------|
| 持久记忆 | ❌ 每次对话重新开始 | ✅ 持久拼图库 |
| 主动融合 | ❌ 用户自己问 | ✅ AI主动找联系 |
| 融合历史 | ❌ 无 | ✅ 可视化融合记录 |
| 碎片增长 | ❌ 无 | ✅ 可视化成长曲线 |
| 行动方案 | ⚠️ 通用建议 | ✅ 基于你的真实碎片 |
| 闭环反馈 | ❌ 无 | ✅ 行动打卡+反馈收集 |

**融合历史可视化（过去3个月）**：
- 展示「你过去3个月融合了15次，输出了3个方向，完成了1个」
- ChatGPT做不到这个

**碎片增长曲线**：
- 展示「你的碎片库从5个增长到30个」
- 让用户看到自己的积累

### 1.7 产品哲学（v1.2新增）

> **「不是告诉你做什么，是让你看到自己已有的力量，然后推你一把。」**

拼图融合引擎的本质不是"AI帮你找副业"（表层的），而是**从内而外的自我成长引擎**。

**核心逻辑**：
1. 外部方案（"你应该做XX"）→ 靠外部驱动力 → 难持续
2. 内部发现（"原来我会的这些拼起来可以做YY"）→ 靠内部驱动力 → 持续

用户不需要被"教会"什么，他们需要的是**看到自己已有的东西能被拼成什么**。看到了，自然想试。试了，自然有反馈。有反馈，自然产生新碎片。有更多碎片，融合更精准。

**这是一个自增强的正向循环**——不是我们推动用户，是用户自己的碎片推动他。

### 1.8 双面结构（v1.2新增）

拼图融合引擎分为两面：

```
第一面（日记/知识库）              第二面（碎片/融合）
━━━━━━━━━━━━━━━━━━━              ━━━━━━━━━━━━━━━━━
📝 每日记录                       🧩 碎片页面
   "今天做了什么/学了什么/             AI自动从日记里检测碎片
    观察到什么"                     碎片入库 + 手动添加
                                  
📚 知识库                         ⚡ 融合引擎
   按时间+标签组织                   点击"融合"→ 8刃切割
   可搜索 + 可关联                   → 输出行动方案
   看到自己的成长轨迹                → 标记完成/反馈
                                  
       ↓ 自动连接 ↓
  每次写完日记
  → AI扫描内容
  → "检测到 3 个可入库碎片"
  → 用户一键确认
  → 碎片进入第二面
```

**为什么要"双面"？**

| 问题 | 只用碎片（单面） | 加日记（双面） |
|------|----------------|--------------|
| 用户不知道录什么碎片 | 冷启动很痛苦 | 写日记比"录碎片"自然得多 |
| 用户不天天打开APP | 只有要融合时才来 | 每天写日记是习惯 |
| 碎片太单薄 | 一句话碎片没有上下文 | 日记是碎片背后的完整故事 |
| 没有"第二大脑"功能 | 碎片库≠知识库 | 日记=真正的知识积累 |

**双面不是两个独立模块，而是一个流程的两端**：
- 第一面是"输入"——你每天自然而然地写，不需要刻意录碎片
- 第二面是"输出"——系统帮你整理、关联、融合、行动

这解决了方案里一直没说清楚的问题：**碎片从哪里来？** 不是用户刻意录入（门槛高），而是日常生活中自然产生（门槛低）。

## 二、核心机制设计

### 2.1 拼图碎片系统（输入层）

#### 碎片类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 🎯 技能 | 会做的事 | Python、写作、视频剪辑 |
| ❤️ 爱好 | 热爱的事 | 钓鱼、reading、徒步 |
| ⚡ 习惯 | 日常自动化行为 | 每天早上写日记、每周健身3次 |
| 💪 能力 | 软技能/底层能力 | 快速学习、说服他人、项目管理 |
| 📚 知识点 | 学到的概念/模型 | 拼图法则、幸运表面积、第二曲线 |
| 🏷️ 经历 | 过往经验 | 开过淘宝店、做过社群运营 |

#### 碎片录入方式

1. **快速录入**：一句话 + AI自动识别类型
   - 用户输入：「我会Python」
   - 系统识别：类型=技能，标签=`#编程`，关联碎片=`自动扫描相似项`
   
2. **批量导入**：从Obsidian/Notion/CSV导入

3. **被动捕获**：用户日常记录时，AI自动提取碎片
   - 用户写日记：「今天用Python自动化处理了100个Excel文件」
   - 系统提示：「检测到新碎片：Python自动化（技能），要入库吗？」

4. **【v1.1新增】预置碎片模板**（冷启动解决方案）
   
   **问题**：需要20-30碎片才能产生有意义的融合，Aha时刻太远
   
   **解决方案**：预置碎片模板
   
   | 职业模板 | 预置碎片（10-15个） | 说明 |
   |---------|-------------------|------|
   | 外卖骑手 | 熟悉城市路线、会骑电动车、每天跑单10小时、会用导航APP、体力好、时间灵活、熟悉商家位置、会简单维修（车子）、抗压能力强、每天接触不同人 | 选模板后，用户确认/修改，立即有10个碎片 |
   | 程序员 | Python、SQL、解决问题能力、逻辑思维、加班习惯、GitHub、算法基础、沟通能力、项目管理、持续学习 | 同上 |
   | 销售 | 说服能力、抗压能力、客户关系管理、目标导向、沟通能力、产品知识、谈判技巧、时间管理 | 同上 |
   | 宝妈 | 时间管理、多任务处理、耐心、教育知识、采购能力、社交网络、财务管理、烹饪技能 | 同上 |
   | 老师 | 知识传授、课程设计、学生管理、沟通能力、学科知识、评估能力、耐心、公开演讲 | 同上 |
   | ... | ... | 持续扩展 |
   
   **使用流程**：
   - 用户注册后，选择职业（或「其他」）
   - 系统自动填充该职业的10-15个常见碎片
   - 用户确认/修改/删除不适合的
   - 立即有10+碎片，第一次融合就有意义
   
   **价值**：冷启动时间从「录入20个碎片」缩短到「选模板+确认5分钟」

5. **【v1.1新增】引导式录入**（降低门槛）
   
   **问题**：用户不知道录什么碎片
   
   **解决方案**：AI问问题，用户回答，AI提取碎片
   
   **流程**：
   ```
   AI: 「你每天做什么工作？」
   用户: 「送外卖，一天跑10小时」
   AI提取: [熟悉城市路线] [体力好] [时间灵活] [抗压能力强]
   
   AI: 「周末喜欢干什么？」
   用户: 「打游戏，也帮朋友代练」
   AI提取: [游戏技能] [代练经验] [服务意识]
   
   AI: 「最近学了什么新东西？」
   用户: 「在看怎么用抖音赚钱」
   AI提取: [短视频运营] [赚钱意识]
   
   AI: 「这些碎片我帮你入库了，确认一下？」
   ```
   
   **价值**：用户不需要「想碎片」，只需要「回答问题」，门槛大幅降低

### 2.2 融合引擎（核心）

#### 触发方式

| 触发方式 | 说明 | 适用场景 |
|---------|------|---------|
| 主动融合 | 用户点击「融合拼图」按钮 | 有灵感/诉求时 |
| 定时融合 | 每周一早上自动融合 | 发现碎片间的隐藏联系 |
| 事件触发 | 新增碎片后，自动检测能否与现有碎片融合 | 入库即触发 |
| 手动指定 | 用户选2-3个碎片，要求融合 | 有明确意图 |

#### 融合流程（映射 linggan v5.2）

```
用户点击「融合拼图」
    ↓
① 收集相关碎片（AI自动选取 + 用户可调整）
    ↓
② 8刃切割（真伪/证伪/实战/失败/笨拙/边界/反向/行动）
    ↓
③ 灵感碰撞（交叉/冲突/嵌套 三模式）
    ↓
④ 洞见质检（认知落差够不够？能验证吗？）
    ↓
⑤ 形态决策（观点卡/方法卡/案例卡/行动清单/口播脚本）
    ↓
⑥ 生成输出（Markdown + PNG卡片 + 可执行文档）
    ↓
⑦ 存入碎片库（新产生的洞见也变成碎片，可被再次融合）
    ↓
⑧ 【v1.1新增】闭环反馈（行动打卡 + 定时推送 + 融合日记）
```

#### 融合输出模板（v1.1优化）

每次融合产出一份**完整文档**，包含：

```markdown
# 融合报告：[标题]

## 🔥 破局点
（1句话，造成认知落差）

## 🗡️ 用了哪些碎片
- 碎片A：[内容]（类型）← 来自[碎片ID]
- 碎片B：[内容]（类型）← 来自[碎片ID]
- 碎片C：[内容]（类型）← 来自[碎片ID]

## 💬 金句
（≤25字，有冲突感）

## 📋 行动清单
- [ ] 1. [具体动作]（今天能做）【可标记完成】
- [ ] 2. [具体动作]（本周能做）【可标记完成】
- [ ] 3. [具体动作]（本月能做）【可标记完成】

## ⚠️ 边界
（什么时候这个融合结论不适用）

## 🔗 下一步融合建议
（基于本次产出，AI推荐下次可以融合的方向）

---
**AI信心度：75%**  ← 【v1.1新增】
**引用来源**：[碎片A]、[碎片B]、[碎片C]  ← 【v1.1新增】
**使用的刃**：真伪刃、实战刃、行动刃  ← 【v1.1新增，高级用户可选】
```

**【v1.1新增】AI输出质量保障机制**：

1. **信心度标注**
   - 融合结果旁边显示：「这个融合方向，AI信心度：75%」
   - 信心度算法：基于碎片数量、碎片类型多样性、历史融合成功率
   - 信心度 < 60%：提示用户「这个方向可能不太准，要不要重新融合？」

2. **引用碎片来源**
   - 输出的每个结论，标注来自哪个碎片
   - 例：「建议做外卖探店」→ 来自[碎片:熟悉城市路线] + [碎片:每天刷短视频]
   - 用户能验证AI不是瞎编

3. **高级用户可选刃**
   - 设置页面：「选择你想要的刃」
   - 例：只想要「行动刃」和「实战刃」，不要「边界刃」
   - 输出更聚焦，减少噪音

#### 【v1.1新增】闭环设计

**问题**：融合完了→看了有道理→然后呢？

**解决方案**：3个闭环功能

1. **行动打卡**
   - 每个行动清单里的项目，用户可以标记「已完成 ✅」
   - 完成后AI追问：「结果怎么样？有什么新发现？」
   - 新发现 → 存入新碎片 → 下一轮融合
   
   **流程**：
   ```
   融合输出：建议做「外卖探店」短视频
   行动1：本周注册抖音账号 ✅（用户标记完成）
   AI追问：「注册完了？下一步打算拍什么内容？」
   用户回答：「拍我送外卖时看到的美食」
   AI提取新碎片：[美食发现能力] [本地餐饮知识]
   → 存入碎片库 → 下次融合可以用
   ```

2. **定时推送**
   - 每周一早上推送：「上周你融合的方向，试了吗？」
   - 没试 → 提醒「要不要换个方向？」或「需要更具体的行动指南吗？」
   - 试了 → 引导反馈 → 新碎片
   
   **推送文案示例**：
   > 早上好！上周你融合出了「外卖探店」方向，试了吗？
   > 
   > - 如果试了，效果怎么样？回复告诉我，我帮你总结新碎片
   > - 如果没试，要不要我帮你拆解成更具体的第一步？
   > - 如果想换个方向，点击重新融合 →

3. **融合日记**
   - 每次融合后，让用户写一句话：「我今天打算怎么用这个？」
   - 下次融合时，AI会引用这句话：「上次你说要...做到了吗？」
   
   **示例**：
   ```
   融合1输出：外卖探店方向
   用户写融合日记：「这周先注册抖音，拍3条测试」
   
   （一周后）
   融合2触发：AI读取融合日记
   AI: 「上次你说要拍3条测试，拍了吗？如果拍了，反馈怎么样？
        如果没拍，是卡在哪了？我们重新融合一个更简单的方向？」
   ```

**闭环的价值**：
- 从「一次性融合」变成「持续改进的循环」
- 用户不是「用完就走」，而是「试了→反馈→再融合」
- 提高留存率 + 提高融合质量（新碎片让后续融合更精准）

### 3.5 数据模型

#### Fragment（碎片）

```sql
CREATE TABLE fragments (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    type VARCHAR(50),  -- 技能/爱好/习惯/能力/知识点/经历
    content TEXT,
    tags TEXT[],  -- PostgreSQL数组类型
    embedding VECTOR(1536),  -- 向量表示，用于相似度检索
    created_at TIMESTAMP,
    last_used_at TIMESTAMP,
    fusion_count INT DEFAULT 0,
    related_ids UUID[],  -- 系统发现的关联碎片
    metadata JSONB  -- 灵活存储额外信息
);
```

#### Fusion（融合记录）

```sql
CREATE TABLE fusions (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    fragment_ids UUID[],  -- 参与融合的碎片
    trigger_type VARCHAR(50),  -- 主动/定时/事件/手动
    cutting_result JSONB,  -- 8刃切割结果
    collision_result JSONB, -- 碰撞结果
    output_type VARCHAR(50), -- 观点卡/方法卡/行动清单/...
    output_content TEXT,
    quality_score INT,  -- 1-5星
    confidence FLOAT,  -- 【v1.1新增】AI信心度（0-1）
    sources UUID[],  -- 【v1.1新增】引用来源碎片ID列表
    used_blades VARCHAR(50)[],  -- 【v1.1新增】使用的刃列表
    created_at TIMESTAMP
);
```

#### 【v1.1新增】CheckIn（打卡记录）

CREATE TABLE checkins (
    id UUID PRIMARY KEY,

    completed BOOLEAN DEFAULT FALSE,
    completed_at TIMESTAMP,

2. ⚡ 一键融合——AI用8种认知刃帮你找联系（标注信心度+引用来源）
3. 📋 可执行输出——不只是洞见，是下一步行动（支持行动打卡+反馈）

### 6.3 增长飞轮设计

```
用户录入碎片（通过模板或引导式，降低门槛）
    ↓
体验融合（Aha时刻，第一次融合就有意义）
    ↓
收到有用输出 → 分享知识卡片到社交媒体
    ↓
新用户通过分享链接注册
    ↓
新用户录入碎片...
    ↓
【v1.1新增】行动打卡 → 反馈 → 新碎片 → 再次融合（闭环）
```

**关键**：分享出去的知识卡片必须带有产品水印/二维码，让收到的人好奇「这个是什么工具生成的？」

## 七、风险与应对（v1.1优化）

### 7.1 技术风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| AI输出质量不稳定 | 高 | 高 | 用用户反馈数据微调；提供「重新融合」按钮；标注信心度 |
| 融合耗时过长（>30s） | 中 | 中 | 异步任务 + 进度条；预计算热门组合 |
| 向量检索不准确 | 中 | 中 | 用更好的embedding模型（如bge-large-zh） |

### 7.2 产品风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 用户不知道录什么碎片 | 高 | 中 | 【v1.1已解决】预置碎片模板 + 引导式录入 |
| 融合输出不够惊艳 | 中 | 高 | MVP阶段重点打磨输出质量；设置质检环节；标注来源让用户验证 |
| 用户留存率低 | 中 | 高 | 【v1.1已解决】定时融合推送（每周一）；行动打卡；融合日记 |

### 7.3 商业风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 大厂复制（Notion/Obsidian） | 低 | 高 | 快速建立品牌 + 用户口碑；专注细分场景（副业探索者） |
| AI成本过高 | 中 | 中 | 换更便宜的模型（DeepSeek）；缓存相似请求 |
| 付费转化率低 | 中 | 高 | 免费版足够好用但有限制；专业版必须有明显增值；新增融合次数包降低门槛 |

## 八、成功指标

### 8.1 North Star Metric（北极星指标）

**每周活跃融合次数**（Weekly Active Fusions, WAF）

这个指标直接反映了产品的核心价值是否被用户感知到。

### 8.2 各阶段目标

| 阶段 | 时间 | 目标 |
|------|------|------|
| MVP内测 | 第2个月末 | 10个内测用户，WAF > 20，行动打卡完成率 > 30% |
| Beta | 第4个月末 | 100个Beta用户，WAF > 300 |
| Launch | 第6个月末 | 500个注册用户，50个付费用户 |
| Growth | 第12个月末 | 5000个注册用户，1000个付费用户 |

### 8.3 关键指标拆解

```
MRR（月经常性收入）= 付费用户数 × ARPU + 融合次数包收入 + 打赏收入
    ↓
需要追踪：
- 注册→激活转化率（目标 > 40%）
- 激活→付费转化率（目标 > 10%）
- 月留存率（目标 > 80%）
- 推荐率（K-factor，目标 > 0.5）
```

## 【v1.1新增】修改记录

### v1.1 优化内容（2026-05-12）

基于深度审视报告，进行以下优化：

1. **目标用户调整**：第一阶段聚焦P1副业探索者（升为P0），P0知识工作者降为P1，P2内容创作者降为P2

2. **冷启动方案**：新增"预置碎片模板"功能
   - 不同职业模板（外卖骑手/程序员/销售/宝妈/老师等）
   - 用户选职业 → 自动填充10-15个常见碎片 → 确认/修改
   - 引导式录入：AI问"你每天做什么？"→提取碎片→确认

3. **闭环设计**：新增3个功能
   - 行动打卡：每个行动可标记完成，AI追问反馈
   - 定时推送：每周一早上"上周方向试了吗？"
   - 融合日记：每次融合后写一句话"今天打算怎么用"

4. **AI输出质量**：新增3个机制
   - 输出时标注AI信心度（百分比）
   - 引用碎片来源（每个结论标注来自哪个碎片）
   - 高级用户可选刃（只想要某些刃的输出）

5. **与ChatGPT竞争**：新增差异化展示
   - 融合历史可视化（过去3个月融合记录）
   - 碎片增长曲线
   - Landing Page加"ChatGPT vs 拼图融合引擎"对比

6. **定价调整**：
   - 免费版：50碎片 + 15次融合/月（当月下载送10次，共25次）
   - 专业版：¥18/月 或 ¥180/年（从¥29降到¥18）
   - 新增：融合次数单独购买 ¥9/10次
   - 新增：打赏功能（¥9/¥18/¥36三档）
   - 删除团队版（Phase 1不需要）

7. **开发顺序调整**：
   - Week 1：碎片模板系统（预置模板 + 引导录入）
   - Week 2：融合引擎核心（8刃切割 + 碎片组合）
   - Week 3：输出渲染（Markdown + PNG卡片）
   - Week 4：闭环功能（行动打卡 + 反馈收集）
   - Week 5-6：测试 + 内测邀请

8. **MVP范围精简**：去掉"碎片管理"和"融合历史"从P1降到Phase 2，MVP只做：
   - 预置碎片模板
   - 引导式录入
   - 手动指定融合
   - 行动清单输出
   - 完成打卡

9. **技术架构更新**：
   - 新增数据表：CheckIn（打卡记录）、FusionDiary（融合日记）、Template（预置模板）
   - 新增API：模板管理、打卡记录、定时推送
   - 前端新增：时间轴、图表、引导式录入界面
