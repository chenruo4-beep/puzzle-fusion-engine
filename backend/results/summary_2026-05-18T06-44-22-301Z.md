首页分层设计建议
核心设计原则：「先安顿自己，再相遇他人」

不做并列Tab，而是做纵深递进——个人旅途在前，合拍入口自然生长于其中。

页面结构建议
┌─────────────────────────────┐
│   你的旅途                   │  ← 主区域，永远在上
│   [碎片卡片流]                │
│   技能 · 经验 · 想法          │
│   + 继续收集                 │
├─────────────────────────────┤
│   ✦ 你已积累 12 个碎片        │  ← 自然过渡带
│   也许，值得找人共同看看       │
│   [发起合拍]  （轻量按钮）     │
├─────────────────────────────┤
│   正在进行的合拍              │  ← 仅有合拍时才出现
│   与 @张明 · 第3天            │
└─────────────────────────────┘
三个关键设计决策

1. 合拍入口不固定，跟随状态出现

碎片数量少时：不显示合拍入口
积累一定数量后：温柔提示"也许值得找人共同看看"
避免空状态下推销付费功能

2. 个人区域要有完整闭环感

碎片收集本身就有成就感（进度、标签、时间轴）
用户不应感到"没有搭档就不完整"
文案用"你的"而非"我的"，保持距离感与尊重

3. 合拍区域用「相遇」而非「功能」语言

❌ 避免	✅ 推荐
开启合拍分析	找人共同看看
立即配对	邀请一个人
解锁行进地图	看看你们能走到哪里
状态驱动的入口策略
新用户           → 只显示个人旅途，合拍入口隐藏
收集5+碎片后     → 出现淡淡的合拍引导
有进行中的合拍   → 合拍区域升至显眼位置
合拍完成付费后   → 行进地图作为主视觉
一句话设计心法

让个人旅途足够好，好到用户自己想分享——合拍是分享欲的自然出口，不是功能的强制路径。
